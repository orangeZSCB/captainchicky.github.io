I think part of the reservation that some people who write numerical code have is that Javascript is designed to be quick to write rather than quick to execute. As such, it may feels like you need to write two copies of the program, one in a language with optimization-amenable semantics for serious use and one for the web in Javascript. I haven’t tried out any of these, but maybe some from this set of languages that can be converted to Javascript might turn out to be more attractive. (I suspect you can’t write just any program in the original language but have to stick to a subset, but it might still be more palatable.)
